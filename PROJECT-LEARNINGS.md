# Project Learnings

Cross-reflection synthesis. This file starts empty and fills as you complete reflections — it's where patterns that show up *more than once* get recorded so you don't relearn them each time.

## How to use it

- After a reflection is delivered, ask: *what did I learn here that would help the next one?* Add it below.
- Keep entries concrete. "Watch the em-dashes in oral drafts" is useful; "be careful with style" is not.
- When a learning has held across two or three reflections, it has earned promotion — fold it into the relevant framework (`frameworks/`) or, if it's a hard rule, into the Constitution. Then it stops being a note and becomes law.
- This is *your* accumulated craft. It is not seeded with anyone else's; it grows from your own work.
