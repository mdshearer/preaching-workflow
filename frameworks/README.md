# Frameworks

Reference docs that shape how a reflection gets developed. Read each one at the stage where it earns its keep — they're not for cover-to-cover reading.

## When to read each

| File | When |
|---|---|
| `voice-profile.md` | Every stage that touches voice — this is *your* voice, created during onboarding. The reference everything is measured against. |
| `voice-style-analysis.md` | Devotional stage start, and during onboarding — the instrument for capturing a voice (what a good voice characteristic looks like) |
| `EXAMPLE-voice-profile.md` | Whenever you want to see what a finished profile looks like — a fictional preacher's, to be replaced with your own |
| `johnson-methodology.md` | Devotional and Hermeneutical stages — sets the posture for inhabiting the text and checks the bridge to today |
| `voice-rubric.md` | Critic stage — the 100-point scoring instrument |
| `stage-templates/` | Starting a new reflection — blank 01–07 files to copy into your reflection folder |

## How they relate

- **`voice-profile.md`** *is* the voice (yours, specific, exemplified). Everything else serves it.
- **`voice-style-analysis.md`** is the *instrument* for building and sharpening a voice profile — the dimensions to capture and the bar a characteristic must clear.
- **`voice-rubric.md`** *scores* the voice (quantitative), at the Critic stage.
- **`johnson-methodology.md`** supplies the *theological frame* — what a faithful reflection is for, independent of voice. (Synthesis of Darrell Johnson; see `../NOTICE.md`.)

## Distinct from PROJECT-LEARNINGS.md

`PROJECT-LEARNINGS.md` at the repo root captures *new* patterns surfaced by your own reflections over time. The frameworks here are the *starting* substrate. Promote a learning into a framework once it has held across two or three reflections.
